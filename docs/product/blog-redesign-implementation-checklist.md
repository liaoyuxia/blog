# 博客体验改版实施清单

## 1. 使用方式

这份清单用于把 [blog-redesign-prd.md](./blog-redesign-prd.md) 落到研发执行。

建议按以下顺序推进：

1. 先完成文案配置和前台结构改版
2. 再补文章详情继续阅读和回访机制
3. 最后处理后台信息架构和编辑流

## 2. 第一阶段：前台价值表达与阅读链路

### 2.1 首页首屏改版

#### 前端任务

- [ ] 重构首页 Hero 模块
- [ ] 增加内容方向标签展示
- [ ] 增加“第一次来先看”模块
- [ ] 将按钮文案替换为“先看精选文章 / 查看全部主题 / 获取更新提醒”
- [ ] 调整首页布局，让推荐内容在首屏或首屏下方自然可见

#### 后端任务

- [ ] 在站点配置中新增首页首屏文案字段
- [ ] 返回首页首屏文案到 `GET /api/profile`
- [ ] 增加“新手推荐”文章配置能力，或在文章接口中支持推荐理由字段

#### 内容配置任务

- [ ] 重写首页定位文案
- [ ] 确定 3 篇“第一次来先看”的文章
- [ ] 为每篇文章补推荐理由

#### 验收

- [ ] 首页首屏不再出现“个人博客”“作品封面”类表达
- [ ] 新用户第一次进入时可以知道站点主题和收益

### 2.2 归档页发现效率优化

#### 前端任务

- [ ] 把当前抽象主题入口改为“主题筛选”
- [ ] 增加排序入口：最新发布 / 最受欢迎 / 适合新读者
- [ ] 在文章卡片中增加“阅读全文”按钮或更明确 CTA
- [ ] 增加推荐理由字段展示

#### 后端任务

- [ ] `GET /api/posts/page` 支持 `sort` 参数
- [ ] 支持 `latest / popular / starter` 三种排序方式
- [ ] 返回推荐理由字段

#### 内容配置任务

- [ ] 为重点文章补“适合谁看”的推荐说明
- [ ] 梳理热门主题顺序

#### 验收

- [ ] 归档页主要筛选入口不再依赖隐藏操作
- [ ] 用户可以在不试错的情况下完成筛选

### 2.3 文章详情继续阅读链路

#### 前端任务

- [ ] 在文章详情页增加结尾总结区
- [ ] 增加相关文章模块
- [ ] 增加上一篇 / 下一篇导航
- [ ] 增加订阅入口
- [ ] 评估文章详情是否从弹层升级为独立路由

#### 后端任务

- [ ] `GET /api/posts/{slug}` 返回 `relatedPosts`
- [ ] `GET /api/posts/{slug}` 返回 `prevPost`
- [ ] `GET /api/posts/{slug}` 返回 `nextPost`
- [ ] 增加相关推荐逻辑：同分类优先、同标签补充、精选兜底

#### 内容配置任务

- [ ] 为文章结尾总结区准备统一文案模板
- [ ] 校验重点文章之间的推荐关系是否合理

#### 验收

- [ ] 正文结束后，继续阅读区立即可见
- [ ] 点赞和评论不再是文章结束后的唯一下一步

### 2.4 最小回访机制

#### 前端任务

- [ ] 首页增加订阅入口模块
- [ ] 文章详情页增加订阅入口
- [ ] 留言成功态增加订阅入口
- [ ] 根据订阅类型展示不同 CTA

#### 后端任务

- [ ] 在站点设置中增加订阅方式字段
- [ ] `GET /api/profile` 返回订阅配置

#### 内容配置任务

- [ ] 确定一期订阅方案：邮件 / RSS / 链接页
- [ ] 配置订阅文案

#### 验收

- [ ] 前台至少有 3 个清晰的回访入口
- [ ] 回访入口文案符合“更新提醒”语义，而不是“关注我”

## 3. 第二阶段：互动转化与后台效率

### 3.1 留言页转化改版

#### 前端任务

- [ ] 提交成功后展示成功反馈区
- [ ] 成功反馈区增加“继续看推荐内容”
- [ ] 成功反馈区增加“获取更新提醒”
- [ ] 留言页增加推荐内容模块

#### 后端任务

- [ ] 留言成功后支持返回推荐内容，或提供热门文章接口

#### 内容配置任务

- [ ] 重写留言页标题与说明
- [ ] 补齐留言成功态文案

#### 验收

- [ ] 用户提交留言后，不会只停留在一句成功提示

### 3.2 后台信息架构重构

#### 前端任务

- [ ] 把后台拆成一级导航：文章 / 站点设置 / 留言 / 评论
- [ ] 默认落地到文章列表页
- [ ] 拆分站点设置页面，避免超长表单默认出现
- [ ] 增加后台首页或任务概览区

#### 后端任务

- [ ] 如有需要，为后台模块拆分接口
- [ ] 支持文章列表筛选字段：状态 / 精选 / 新手推荐

#### 验收

- [ ] 后台首次登录后，主要任务入口明显可见
- [ ] 默认页面不再是低频配置表单

### 3.3 后台编辑流优化

#### 前端任务

- [ ] 取消“编辑文章前先确认”的弹窗
- [ ] 点击编辑后直接进入编辑态
- [ ] 保存文章后保留编辑上下文
- [ ] 增加草稿保存
- [ ] 增加预览入口

#### 后端任务

- [ ] 确保草稿状态接口可用
- [ ] 如需预览，补充预览数据结构

#### 验收

- [ ] 编辑流程减少不必要确认
- [ ] 保存后不强制打断当前操作
