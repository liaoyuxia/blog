# 博客体验改版 PRD

## 1. 文档信息

- 文档类型：产品需求文档（PRD，产品需求说明）
- 适用范围：前台内容体验、回访机制、后台运营效率
- 当前版本：v1.0
- 目标：把当前站点从“展示型博客”升级为“内容产品型站点”，重点提升首屏理解、持续阅读、回访率和内容运营效率

## 2. 项目背景

当前站点已经具备以下基础能力：

- 前台首页、文章归档、文章详情弹层、留言页
- 后台登录、站点设置、文章管理、留言管理、评论管理
- 双语文案、搜索、分类筛选、标签筛选、点赞、评论、留言

当前主要问题不在于“功能不能用”，而在于“用户路径没有形成闭环”：

- 首页更像作者展示，而不是读者价值说明
- 文章详情读完即结束，缺少继续阅读承接
- 用户喜欢内容后，没有订阅、收藏、提醒等回访机制
- 留言页承接了互动，但没有把互动转回内容浏览
- 后台默认进入超长单页，内容维护效率偏低

## 3. 改版目标

### 3.1 业务目标

- 提高新用户在 3 秒内理解站点价值的概率
- 提高单次访问的阅读深度
- 建立最小可用的回访机制
- 提升后台运营效率，支撑更稳定的内容更新频率

### 3.2 产品目标

- 从“作者自述型表达”切换为“读者收益型表达”
- 从“单篇阅读”升级为“连续阅读路径”
- 从“只有互动”升级为“互动 + 继续阅读 + 回访承接”
- 从“配置中心式后台”升级为“任务导向式后台”

## 4. 目标用户

### 4.1 新访客

- 第一次进入站点
- 需要快速判断站点是否值得继续看
- 不愿意花时间主动探索复杂入口

### 4.2 轻度读者

- 已经点开一篇内容
- 对内容有兴趣，但需要被明确引导继续阅读
- 可能会点赞、评论，但尚未形成稳定回访

### 4.3 潜在回访用户

- 对内容方向认可
- 愿意接收后续更新提醒
- 需要低成本、低打扰的订阅入口

### 4.4 管理员

- 需要高频发布文章、修改内容、处理留言评论
- 不希望每次进入后台都先面对超长设置表单

## 5. 成功指标

### 5.1 前台核心指标

- 首页首屏点击率：首页首屏按钮点击率 >= 当前版本基线的 1.5 倍
- 首页到文章页转化率：提升 30%
- 单次访问平均浏览页数：提升 25%
- 文章详情继续阅读点击率：新增指标，目标 >= 20%
- 文章详情退出率：下降 20%
- 回访入口点击率：新增指标，目标 >= 8%
- 7 日回访率：新增指标，目标逐期提升

### 5.2 后台效率指标

- 管理员首次登录后找到“新建文章/编辑文章”时间 < 10 秒
- 单篇文章编辑完成时间下降 20%
- 发文/改文过程中上下文打断次数下降

## 6. 改版原则

### 6.1 表达原则

- 先讲用户能获得什么，再讲作者是谁
- 少讲“我在记录什么”，多讲“你能学到什么”
- 少用“个人博客”“作品封面”“作者气质”这类自我展示表达

### 6.2 交互原则

- 用户完成任何一步动作后，都要看到下一步
- 内容产品优先强调“继续阅读”，互动是辅助，不是终点
- 搜索、筛选、推荐等发现能力要显性，不让用户猜

### 6.3 运营原则

- 回访机制先上最小可用版，不追求一步到位
- 后台优先支撑高频任务，不让低频配置占据默认入口

## 7. 范围定义

### 7.1 本期范围

- 首页首屏信息架构改版
- 首页“第一次来先看”内容导览
- 文章详情继续阅读链路
- 最小回访机制
- 归档页筛选与内容发现优化
- 留言页互动转化优化
- 后台信息架构和编辑流优化
- 文案体系整体重写

### 7.2 非本期范围

- 用户账户体系
- 个性化推荐算法
- 完整通知中心
- 多作者协作系统
- 复杂会员或付费能力

## 8. 信息架构调整

### 8.1 前台一级结构

- 首页：建立价值认知，提供起步入口和精选内容
- 文章与专题页：承接系统化浏览、筛选和发现
- 文章详情页：承接深度阅读和继续阅读
- 交流与反馈页：承接互动，并回流到内容和订阅

### 8.2 后台一级结构

- 文章
- 站点设置
- 留言
- 评论

默认进入“文章”模块，而不是“站点设置”模块。

## 9.5 后台

### 9.5.1 目标

- 降低后台信息过载
- 提升发文和改文效率

### 9.5.2 信息架构

- 一级导航：文章、站点设置、留言、评论
- 默认落地页：文章列表

### 9.5.3 文章模块

#### 列表字段

- 标题
- 状态
- 分类
- 发布时间
- 浏览量
- 点赞量
- 评论量
- 是否精选
- 是否新手推荐

#### 功能

- 新建文章
- 编辑文章
- 删除文章
- 筛选：已发布 / 草稿 / 精选 / 新手推荐
- 排序：最新、最热

#### 编辑流程优化

- 点击编辑后直接进入编辑态
- 不再先弹确认框
- 编辑后保留当前上下文，不强制滚动到顶部

### 9.5.4 站点设置模块

按模块拆分，不再以超长表单默认呈现：

- 基础信息

- 页脚文案

### 9.5.5 留言和评论模块

- 支持按时间排序
- 支持未处理/已处理筛选
- 支持快速删除
- 预留“标记已读”能力

## 10. 文案体系重写

## 10.1 文案方向

站点所有核心文案从“自我展示”改为“读者价值”。

### 不建议继续使用的表达

- 个人博客
- 记录我的过程
- 作品封面
- 建立作者气质
- 独立开发者自我展示

### 推荐表达

- 内容系统
- 技术实践
- 阅读体验
- 搭建方法
- 运营方法
- 可直接参考的方案

## 10.2 重点替换清单

| 场景 | 旧表达 | 新表达 |
| --- | --- | --- |
| 首页定位 | 基于 React 与 Spring Boot 搭建的正式版博客 | 内容系统、技术实践与阅读体验方法 |
| 首页副标题 | 围绕个人博客、内容平台与前端体验，记录一个正式项目从信息架构到工程落地的过程 | 聚焦内容站点的结构设计、后台实现、阅读体验和持续运营方法，帮助你更快完成搭建与优化 |
| 首页按钮 | 浏览文章 | 先看精选文章 |
| 首页按钮 | 留下反馈 | 交流问题 |
| 关于标题 | 一个更像作品封面的首页 | 你能在这里获得什么 |
| 归档页标题 | 文章 | 文章与专题 |
| 留言页标题 | 留言 | 交流与反馈 |
| 留言页说明 | 欢迎留下建议、感受或合作想法 | 如果你对内容结构、后台实现或阅读体验优化有问题，可以直接在这里交流 |
| 后台标题 | 内容管理与留言管理 | 内容发布与站点运营 |

## 14. 验收标准

- 首页首屏可在 3 秒内说明站点内容价值
- 首页存在明确的新用户起步入口
- 文章详情读完后至少有 2 条继续阅读路径
- 首页、详情页、留言成功态都有回访入口
- 后台登录后优先进入文章相关任务
- 后台编辑流程中不再出现明显的上下文打断
